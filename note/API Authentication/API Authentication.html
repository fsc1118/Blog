API Authentication

Why APIs are authenticated

APIs are authenticated to prevent unauthorized access to the API.

For example, here is the API transferring money: /transfer?from=A&to=B&amount=1000

If the API is not authenticated, anyone can access the API and transfer money from A to B. The point of API authentication is to ensure, in this case, request made by people other than A will be rejected.

API Authentication Methods

1. Password Authentication

When calling API, username and password are embeded in the HTTP body for application to verify.

2. Token Authentication

When calling API, each user has a unique token that identifies the user. When client sends request, that token is embeded in the HTTP body. The request is accepted if the token is valid.

Token should be refreshed from time to time.

3. OAuth

OAuth is a protocol that grants application a token to fetch a user's information such as email address.

Application can use this token to authenticate the user similar to Token Authentication mentioed above. Client will store the token which is passed to application to identify a user.

OAuth has the merit that application doesn't need to explicitly handle the token such as storing and refreshing it.
